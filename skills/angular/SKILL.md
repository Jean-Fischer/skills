# Angular Standards

## Purpose
Use this skill for Angular application code.

## Core rules
1. Follow the repository’s existing Angular style first.
2. Prefer the local architectural pattern when one already exists.
3. Keep components focused and templates simple.
4. Prefer modern Angular patterns when the project baseline supports them.
5. Keep changes small, readable, and easy to validate.

## Workflow
1. Inspect the existing code.
2. Match the local Angular pattern.
3. Make the smallest useful change.
4. Check related templates, styles, and services.
5. Validate the result.

## Validation
Use the repository’s existing build and test commands.

See `HANDBOOK.md` for the fuller Angular guidance.
