# Angular Test Standards

## Purpose
Use this skill for Angular test code.

## Core rules
1. Test behavior, not internals.
2. Follow the repository’s existing testing style.
3. Keep tests readable and stable.
4. Prefer the project’s current Angular testing tools.
5. Keep test setup small and visible.

## Workflow
1. Inspect the existing test style.
2. Match the local testing pattern.
3. Add or update focused tests.
4. Remove brittle assertions when needed.
5. Validate the result.

## Validation
Use the repository’s existing test command.

See `HANDBOOK.md` for the fuller Angular testing guidance.
